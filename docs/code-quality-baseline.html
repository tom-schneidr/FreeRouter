Application Construction Paths

  • FastAPI server — app/main.py lifespan builds StateManager, ModelCatalog, providers, GatewayRequestLimiter, APILiveMonitor, httpx.AsyncClient, WaterfallRouter, EndpointDiagnosisService, optional BackgroundEndpointDiagnosis; stores each on app.state.
  • Programmatic client — app/client.py UnifiedAIClient duplicates settings, state, catalog, and router construction without diagnosis, live monitor, or limiter.
  • Desktop sidecar — Tauri launches packaged uvicorn via sidecar; desktop token env gates /v1/desktop/*.
  • Tray launcher — app/tray_launcher.py opens browser to legacy /chat and /live pages.

Architectural Dependency Concerns

  • Core gateway (app/main.py, router.py) imports UI modules (app/ui/*, app/legacy_pages) for HTML routes.
  • Duplicate service construction between main.py lifespan and client.py.
  • Streaming logic split across openai_stream_routing.py, stream_route.py, and large handlers in main.py.
  • Route selection policy duplicated between router._iter_route_events_with_client and openai_stream_routing.waterfall_openai_stream.
  • state.py and model_catalog.py mix domain rules with persistence I/O.
  • No typed application container; route handlers cast untyped app.state attributes.

Roadmap Risks

  • Legacy HTML pages remain embedded in React (LEGACY_EMBED_SECTIONS); premature removal breaks Chat, Usage, Route Health, and Live Traffic.
  • Tray launcher still opens /chat and /live directly.
  • Large modules exceed soft targets (main 1203, state 1025, model_catalog 1209 lines).
  • Endpoint diagnosis auto-maintenance can mutate catalog; changes need characterization tests.
  • Streaming release-slot setting affects limiter lifecycle during long streams.
